## 第一部分：Keil5 环境搭建（小白首选，推荐）

> **为什么先用 Keil5？**  
> - 你学长/老师大概率用它，出问题好问人  
> - 图形界面操作，不用记命令行  
> - STM32CubeF4 自动生成工程骨架  
> - VSCode 等熟悉了再切换

---

### 1.1 安装 Keil5 MDK-ARM

**第一步：下载安装包**

1. 打开 https://www.keil.com/download/product/
2. 找到 "MDK-ARM" 一栏，点 **Download** 按钮
3. 填写姓名、邮箱等信息后开始下载
4. 下载的文件名类似 `MDK538.EXE`（版本号可能不同）

**第二步：安装**

1. 双击 `MDK538.EXE`，一路点 **Next**
2. ⚠️ 安装路径建议用默认：`C:\Keil_v5\`
3. 中间会弹出一个窗口让你填 Name / Company / E-mail — **随便填**就行
4. 安装最后一步勾选 "Launch the Pack Installer"
5. 完成

**第三步：激活（可以用社区版/评估版）**

- 如果是学校机房：一般已经激活好了
- 如果是自己电脑：Keil 有 32KB 限制的免费评估版，课程设计够用（我们的代码优化后可能超 32KB，需要注册）
- 有的学校购买了 Keil 教育许可，去学校软件站下载激活工具

---

### 1.2 安装 STM32F4 芯片包

1. 安装完 Keil5 后，会自动打开 **Pack Installer**（如果没有，在 Keil5 菜单栏点那个小芯片图标）
2. 在左侧找到 **STMicroelectronics → STM32F4 Series → STM32F407 → STM32F407VG**
3. 点击右侧的 **Install** 按钮
4. 等待下载安装完成（可能需要几分钟，看网络速度）

> 如果 Pack Installer 下载很慢或连不上：
> - 直接去 https://www.keil.com/dd2/pack/ 下载 `.pack` 文件
> - 双击 `.pack` 文件即可导入

---

### 1.3 获取 STM32CubeF4 库文件

> ⚠️ **这是最容易出错的一步，请仔细阅读！**

STM32CubeF4 包含了 HAL 库（硬件抽象层）和 CMSIS（ARM 内核接口），是 STM32 编程的基础。

**方法一：从 ST 官网下载（推荐）**

1. 打开 https://www.st.com/en/embedded-software/stm32cubef4.html
2. 点击 **"Get Software"** 按钮
3. 注册 ST 账号（免费，用邮箱注册）
4. 下载 `en.stm32cubef4.zip`
5. 解压到桌面或其他位置。解压后文件夹结构大概是：

```
STM32Cube_FW_F4_V1.xx.x/
├── Drivers/
│   ├── CMSIS/
│   │   ├── Core/Include/           ← core_cm4.h, cmsis_gcc.h 等
│   │   ├── Device/ST/STM32F4xx/
│   │   │   ├── Include/            ← stm32f407xx.h 等
│   │   │   └── Source/Templates/   ← system_stm32f4xx.c
│   │   └── ...
│   └── STM32F4xx_HAL_Driver/
│       ├── Inc/                    ← 所有 HAL 头文件
│       └── Src/                    ← 所有 HAL 源文件
├── Middlewares/
│   └── Third_Party/
│       └── FreeRTOS/               ← FreeRTOS 源码！！！在 CubeF4 包里就有
│           └── Source/
│               ├── include/        ← FreeRTOS.h, task.h 等
│               ├── CMSIS_RTOS_V2/  ← cmsis_os2.h, cmsis_os2.c
│               ├── portable/       ← 平台适配代码
│               │   ├── GCC/ARM_CM4F/    ← GCC 编译器适配
│               │   ├── RVDS/ARM_CM4F/   ← Keil 编译器适配 ← 我们用这个！
│               │   └── MemMang/         ← 内存管理 (heap_4.c)
│               ├── croutine.c
│               ├── tasks.c
│               ├── queue.c
│               ├── ...
│               └── timers.c
└── Projects/
```

> 🎉 **好消息**：STM32CubeF4 包里**自带 FreeRTOS**！你不用去别的地方下载。路径在 `Middlewares/Third_Party/FreeRTOS/`

---

### 1.4 验证你需要的文件

打开解压后的 `STM32Cube_FW_F4_V1.xx.x` 文件夹，确认你能找到以下文件：

```
✅ Drivers/CMSIS/Core/Include/core_cm4.h
✅ Drivers/CMSIS/Device/ST/STM32F4xx/Include/stm32f407xx.h
✅ Drivers/CMSIS/Device/ST/STM32F4xx/Source/Templates/system_stm32f4xx.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal_tim.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal_uart.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal_gpio.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal_rcc.c
✅ Drivers/STM32F4xx_HAL_Driver/Src/stm32f4xx_hal_i2c.c
✅ Middlewares/Third_Party/FreeRTOS/Source/tasks.c
✅ Middlewares/Third_Party/FreeRTOS/Source/queue.c
✅ Middlewares/Third_Party/FreeRTOS/Source/CMSIS_RTOS_V2/cmsis_os2.c
```

每找到一项就打勾 ✅。如果某个文件找不到，说明你的 CubeF4 包版本不一样，在文件夹里搜一下文件名。

## 第二部分：Keil5 工程搭建（手把手）

### 2.1 整理库文件目录

我们的项目已经放在了 `code/STM32/` 下面。现在需要把 STM32CubeF4 的库文件和 FreeRTOS 也放进来。

**操作步骤**（在文件资源管理器中操作）：

```
1. 打开 f:\KeShe\code\STM32\

2. 创建两个空文件夹:
   - f:\KeShe\code\STM32\Drivers\
   - f:\KeShe\code\STM32\Middlewares\

3. 把 STM32CubeF4 包中的内容复制过来：

   FROM: STM32Cube_FW_F4_V1.xx.x\Drivers\CMSIS\
   TO:   f:\KeShe\code\STM32\Drivers\CMSIS\

   FROM: STM32Cube_FW_F4_V1.xx.x\Drivers\STM32F4xx_HAL_Driver\
   TO:   f:\KeShe\code\STM32\Drivers\STM32F4xx_HAL_Driver\

   FROM: STM32Cube_FW_F4_V1.xx.x\Middlewares\Third_Party\FreeRTOS\
   TO:   f:\KeShe\code\STM32\Middlewares\Third_Party\FreeRTOS\
```

> **注意**：文件夹名字必须一模一样，包括大小写。否则后面头文件引用会报错。

最终你的 `code/STM32/` 目录应该长这样：

```
code/STM32/
├── Drivers/
│   ├── CMSIS/
│   │   ├── Core/Include/          ← 从 CubeF4 复制
│   │   └── Device/ST/STM32F4xx/  ← 从 CubeF4 复制
│   └── STM32F4xx_HAL_Driver/
│       ├── Inc/
│       └── Src/
├── Middlewares/
│   └── Third_Party/
│       └── FreeRTOS/
│           └── Source/            ← 从 CubeF4 复制
├── Inc/                           ← 我们写的头文件（已有）
├── Src/                           ← 我们写的源文件（已有）
├── .vscode/                       ← VSCode 配置（已有）
├── Makefile                       ← ARM GCC 编译脚本（已有）
└── VSCode调试指南.md               ← VSCode 教程（已有）
```

### 2.2 创建 Keil5 工程

1. 打开 **Keil5**
2. 菜单栏：**Project → New μVision Project...**
3. 导航到 `f:\KeShe\code\STM32\` 文件夹
4. 文件名填 `smart_carrier`，点保存
5. 弹出芯片选择窗口：展开 **STMicroelectronics → STM32F4 Series → STM32F407 → STM32F407VG**，选中，点 OK
6. 弹窗 "Manage Run-Time Environment"：直接点 **Cancel**（我们会手动添加所有文件）

现在你有了一个空的 Keil 工程。

### 2.3 导入源文件和头文件路径

Keil5 的工程窗口有 4 个面板（在左侧垂直排列）：

| 图标 | 名称 | 用途 |
|------|------|------|
| 📁 | Project | 文件管理（添加/删除源文件） |
| 📖 | Books | 帮助文档 |
| 🔧 | Functions | 函数列表 |
| 📋 | Templates | 代码模板 |

我们主要在 **Project** 面板操作。

**第一步：创建文件分组**

在 Project 面板右键 `Target 1` → **Manage Project Items...**

在弹出的窗口中，在 "Groups" 栏点 **新建** 按钮（小方块图标），依次创建以下分组：

```
📁 Application     ← 我们的应用代码
📁 HAL_Driver      ← STM32 HAL 库
📁 CMSIS           ← ARM CMSIS 内核
📁 FreeRTOS_Core   ← FreeRTOS 内核
📁 FreeRTOS_Port   ← FreeRTOS 平台适配
```

效果如下：

```
Target 1
├── Application
├── HAL_Driver
├── CMSIS
├── FreeRTOS_Core
└── FreeRTOS_Port
```

### 2.4 工程设置（关键！）

点 **Project → Options for Target 'Target 1'**（或点工具栏上的魔术棒图标 🪄）。

#### Tab 1: Target

| 选项 | 值 |
|------|-----|
| Xtal (MHz) | 8.0 |
| Operating System | **FreeRTOS** |
| Read/Write Memory Areas | ✅ 保持默认 |

#### Tab 2: Output

| 选项 | 值 |
|------|-----|
| ✅ Create HEX File | 勾选 |
| Select Folder for Objects... | 新建一个 `build/` 文件夹 |

#### Tab 3: C/C++ (重点！)

| 选项 | 值 |
|------|-----|
| **Define** | `USE_HAL_DRIVER,STM32F407xx` |
| **Optimization** | Level 0 (-O0) ← 调试阶段一定用 O0 |
| **One ELF Section per Function** | ✅ 勾选 |
| **C99 Mode** | ✅ 勾选 |
| **GNU extensions** | ✅ 勾选 |

**Include Paths** — 点击右边的 `...` 按钮，添加以下路径：

```
.\Inc
.\Drivers\CMSIS\Core\Include
.\Drivers\CMSIS\Device\ST\STM32F4xx\Include
.\Drivers\STM32F4xx_HAL_Driver\Inc
.\Middlewares\Third_Party\FreeRTOS\Source\include
.\Middlewares\Third_Party\FreeRTOS\Source\CMSIS_RTOS_V2
.\Middlewares\Third_Party\FreeRTOS\Source\portable\RVDS\ARM_CM4F
```

> 用 `.\` 开头的相对路径，不要用 `f:\KeShe\...` 绝对路径，这样换电脑工程也能用。

#### Tab 4: Debug

| 选项 | 值 |
|------|-----|
| Use | **ST-Link Debugger** |
| Settings → Port | **SW** |
| Settings → Connect | **Normal** |
| Settings → Reset | **Autodetect** |

#### Tab 5: Utilities

| 选项 | 值 |
|------|-----|
| ✅ Use Debug Driver | 勾选 |
| ✅ Update Target before Debugging | 勾选 |
| Settings → Flash Download → Add | **STM32F4xx 1MB Flash** |

#### 关于 `stm32f4xx_hal_conf.h`

HAL 库需要一个配置文件来告诉它你用了哪些模块。从 CubeF4 模板复制：

```
FROM: STM32Cube_FW_F4_V1.xx.x\Drivers\STM32F4xx_HAL_Driver\Inc\stm32f4xx_hal_conf_template.h
TO:   f:\KeShe\code\STM32\Inc\stm32f4xx_hal_conf.h
```

然后打开这个文件，确保以下宏是 **启用的**（去掉注释符号）：

```c
#define HAL_MODULE_ENABLED
#define HAL_CORTEX_MODULE_ENABLED
#define HAL_GPIO_MODULE_ENABLED
#define HAL_RCC_MODULE_ENABLED
#define HAL_TIM_MODULE_ENABLED
#define HAL_UART_MODULE_ENABLED
#define HAL_I2C_MODULE_ENABLED
#define HAL_DMA_MODULE_ENABLED
#define HAL_PWR_MODULE_ENABLED
#define HAL_FLASH_MODULE_ENABLED
```

### 3.1 MaixCAM Pro 开箱与基础配置

**第一步：连接 MaixCAM 到电脑**

1. 用 Micro USB 线连接 MaixCAM 和电脑
2. MaixCAM 开机后，电脑会识别为一个 U 盘 + 串口
3. 如果没识别 U 盘 → 安装 CH340/CH342 驱动

**第二步：使用 MaixPy IDE（最简单的上传方式）**

1. 打开浏览器，访问 MaixPy IDE 在线版：https://wiki.sipeed.com/maixpy → 找到 IDE 链接
2. 或者下载桌面版 MaixPy IDE

**第三步：确认通信正常**

1. 在 IDE 中选择正确的串口号（设备管理器里看）
2. 点击连接（波特率 115200）
3. 在终端输入 `print("hello")` → 如果有输出说明连接成功

### 3.2 上传视觉代码

将 `code/MaixCAM/` 下的 5 个 Python 文件复制到 MaixCAM 的 U 盘（或通过 IDE 上传）：

1. `config.py` — 配置参数
2. `qr_detector.py` — QR 码检测
3. `color_detector.py` — 颜色识别
4. `comm.py` — UART 通信
5. `main.py` — 主程序

在 MaixCAM 的文件系统中，将 `main.py` 设为开机自动运行（放到 `boot.py` 旁边）。

> 具体上传方法请参考 MaixCAM 官方文档：https://wiki.sipeed.com/maixcam-pro

## 第四部分：硬件连接

### 4.1 ST-Link 连接 STM32

STM32F407 的 SWD 调试接口需要 4 根线：

```
ST-Link              STM32F407 核心板
┌────────┐           ┌──────────┐
│ GND   ├───────────┤ GND      │
│ SWCLK ├───────────┤ PA14/SWCLK│
│ SWDIO ├───────────┤ PA13/SWDIO│
│ 3.3V  ├───────────┤ 3.3V     │  ← 可选，如果核心板已独立供电
└────────┘           └──────────┘
```

> ⚠️ **接线顺序**：先接 GND，再接信号线。不要在通电状态下插拔！

### 4.2 MaixCAM ↔ STM32 UART 连接

```
MaixCAM              STM32F407
┌────────┐           ┌──────────┐
│ TX     ├───────────┤ PB11/RX  │  MaixCAM 发送 → STM32 接收
│ RX     ├───────────┤ PB10/TX  │  STM32 发送 → MaixCAM 接收
│ GND    ├───────────┤ GND      │  ← 必须共地！
│ 5V     ├───────────┤ 5V       │  ← 可选，如果 MaixCAM 需要供电
└────────┘           └──────────┘
```

### 4.3 各传感器和执行器连接

引脚分配详见 [pin_config.h](code/STM32/Inc/pin_config.h)。简单总结：

| 设备 | 引脚 | 说明 |
|------|------|------|
| 电机1 (左前) PWM | PA0 | TIM2_CH1 |
| 电机2 (右前) PWM | PA1 | TIM2_CH2 |
| 电机3 (左后) PWM | PA2 | TIM2_CH3 |
| 电机4 (右后) PWM | PA3 | TIM2_CH4 |
| 舵机1 (腰部) | PE9 | TIM1_CH1 |
| 舵机2 (大臂) | PE11 | TIM1_CH2 |
| 舵机3 (小臂) | PE13 | TIM1_CH3 |
| 舵机4 (手爪) | PE14 | TIM1_CH4 |
| 灰度传感器 5路 | PD8~PD12 | GPIO 输入 |
| 超声波 TRIG | PB0 | GPIO 输出 |
| 超声波 ECHO | PB1 | GPIO 输入 |
| OLED I2C | PB8(SCL)/PB9(SDA) | I2C1 |
| 启动按钮 | PE0 | GPIO 输入 |
| 蜂鸣器 | PB8 | GPIO 输出 |

> 📌 轮趣科技麦轮小车的核心板一般已经接好电机驱动和灰度传感器，舵机和超声波需要自己接。

## 第五部分：烧录与调试

### 5.1 用 Keil5 烧录

1. 连好 ST-Link，小车通电
2. Keil5 中按 **F7** 编译
3. 按 **F8**（或工具栏 **Load** 按钮）下载
4. 底部输出窗口显示 `Flash Load finished` 即烧录成功
5. 按小车上的复位按钮或重新上电，程序开始运行

### 5.2 用 Keil5 调试

**启动调试**：

1. 编译通过后，按 **Ctrl+F5**（或工具栏放大镜🔍图标）
2. Keil5 进入调试模式，程序停在 `main()` 第一行
3. 此时可以看变量、设断点、单步执行

**调试窗口布局**：

```
┌─────────────────────────────────────────────────────┐
│  菜单栏 / 调试工具栏                                  │
│  [R] [H] [Step] [StepOver] [StepOut] [RunToCursor] │
├────────────┬────────────────────────────────────────┤
│  源代码窗口  │  可以设断点（点行号左边）                 │
│             │  main.c, task_manager.c ...            │
├────────────┼────────────────────────────────────────┤
│ Watch 窗口  │  Call Stack 窗口  │  Memory 窗口        │
│  查看变量值  │   函数调用栈       │   查看内存内容       │
└────────────┴────────────────────────────────────────┘
```

**快捷键**：

| 按键 | 功能 | 说明 |
|:---:|------|------|
| `F5` | 运行 | 全速运行，遇到断点停止 |
| `F11` | 单步进入 | 进入函数内部 |
| `F10` | 单步跳过 | 执行当前行，不进函数 |
| `Ctrl+F11` | 单步跳出 | 执行完当前函数并返回 |
| `F9` | 断点 | 在当前行设置/取消断点 |
| `Ctrl+Shift+F5` | 重启 | 重新开始调试 |
| `Esc` | 停止 | 退出调试模式 |

**查看变量**：

1. 在 Watch 窗口右键 → **Add Item**
2. 输入变量名，如 `g_sys.state`
3. 运行时会实时显示该变量的值

**常用调试变量**：

| 变量名 | 含义 |
|--------|------|
| `g_sys.state` | 当前状态机状态 (0~18) |
| `g_sys.current_step` | 当前搬运步骤 (0~5) |
| `g_sys.task_code.digits[0]` | 任务编码第1位 |
| `g_motors[0].current_rpm` | 左前电机实时转速 |
| `g_motors[0].target_rpm` | 左前电机目标转速 |
| `g_chassis.pose.x_mm` | 当前 X 坐标 |
| `g_chassis.pose.y_mm` | 当前 Y 坐标 |
| `g_sys.has_error` | 是否出错 |

**查看外设寄存器**（类似 VSCode 的 SVD 视图）：

Keil5 菜单：**Peripherals → System Viewer** → 选择外设（如 TIM2, GPIOA）

### 5.3 常用调试技巧

**技巧1：加断点看流程走到哪了**

在 `task_manager.c` 的每个状态入口加断点：

```c
case STATE_SCAN_QR:
    {   // ← F9 在这一行设断点
        CommError_t err = ...
```

运行后看程序停在哪个状态，就知道当前流程走到哪了。

**技巧2：看 FreeRTOS 任务是否都在运行**

在 `Task_Monitor` 里 `osDelay(500)` 处设断点，看是否能定期到达。

**技巧3：用蜂鸣器打信号**

如果某些地方不方便加断点（比如电机高速运转时），可以在代码里加：

```c
TaskManager_Beep(100);  // 响一声表示执行到这里
```

**技巧4：用 OLED 输出调试信息**

我们的 OLED 一直在显示当前状态。如果机器人跑远了看不到串口，看 OLED 就行。

**技巧5：printf 调试（需要重定向）**

在 `main.c` 加一个 `printf` 重定向函数，通过 ST-Link 的 SWO 或 UART 输出调试信息。这里不做要求，有需要再搜索 "STM32 printf retarget"。

## 第六部分：适配你的机器人

克隆代码后不能直接跑！每个机器人的硬件参数（电机极性、舵机角度、颜色阈值、场地坐标）都需要现场标定。以下是必须修改的内容。

### 6.1 最常需要修改的参数

打开 [pin_config.h](code/STM32/Inc/protocol.h)，修改这些宏：

```c
// 场地坐标 — 根据实际赛场测量
#define QR_X_MM                 400    // 二维码板的X坐标
#define QR_Y_MM                 0      // 二维码板的Y坐标
#define RAW_AREA_X_MM           0      // 原料区X
#define RAW_AREA_Y_MM           600    // 原料区Y
#define ROUGH_AREA_X_MM         0      // 粗加工区X
#define ROUGH_AREA_Y_MM        -300    // 粗加工区Y
#define TEMP_AREA_X_MM          0      // 暂存区X
#define TEMP_AREA_Y_MM         -700    // 暂存区Y
```

> 以上坐标仅供参考！必须在实际赛场上用卷尺量！

### 6.2 PID 参数调节

打开 [motor.c](code/STM32/Src/motor.c)：

```c
#define MOTOR_KP    0.08f   // ← 如果电机抖动，减小 KP
#define MOTOR_KI    0.02f   // ← 如果到达目标慢，增大 KI
#define MOTOR_KD    0.01f   // ← 如果超调大，增大 KD
```

**简单调参口诀**：
- 电机"抽风"（抖动/啸叫） → 减小 KP
- 速度到不了目标 → 增大 KI
- 到目标后冲过头 → 增大 KD

### 6.3 颜色阈值标定

打开 [config.py](code/MaixCAM/config.py)：

```python
# 红色 — 这两个阈值需要根据实际光照调整
RED_THRESHOLD_1 = (0, 80, 20, 80, 10, 80)
RED_THRESHOLD_2 = (0, 80, -80, -10, 10, 80)

# 绿色
GREEN_THRESHOLD = (0, 80, -80, -20, -10, 20)

# 蓝色
BLUE_THRESHOLD  = (0, 60, -20, 30, -80, -20)
```

标定方法（在 MaixCAM 上操作）：

1. 跑 MaixPy 阈值调节工具
2. 把红/绿/蓝物料放在相机视野中
3. 调节 L/A/B 的最小最大值，直到画面中物料区域被完整选中
4. 记录阈值，填回 `config.py`

详细步骤见 MaixPy 官方文档：https://wiki.sipeed.com/maixpy

### 6.4 舵机角度校准

打开 [servo.c](code/STM32/Src/servo.c)，找到预设姿态表：

```c
static const uint16_t g_pose_angles[ARM_POSE_NUM][SERVO_NUM] = {
    [ARM_POSE_IDLE]       = { 90, 20, 10,  5 },   // ← 修改这些角度值
    [ARM_POSE_SCAN]       = { 90, 60, 30,  5 },
    [ARM_POSE_PRE_PICK]   = { 90, 80, 60, 60 },
    // ...
};
```

校准方法：
1. 先用 Keil5 调试模式，在 `Task_Arm` 里手动调一个姿态
2. 看机械臂实际到达的位置
3. 如果手臂太高/太低 → 调整 `SERVO_SHOULDER`（第二位）
4. 如果手爪没抓牢 → 调小 `SERVO_GRIPPER`（第四位）的闭合角度

## 第八部分：常见问题排查

### Q1: 编译错误 `stm32f4xx_hal.h: No such file or directory`

**原因**：头文件路径没配好。

**解决**：
1. 确认 `stm32f4xx_hal.h` 确实在 `Drivers\STM32F4xx_HAL_Driver\Inc\` 下面
2. Keil5 → 魔术棒 → C/C++ → Include Paths → 检查 `.\Drivers\STM32F4xx_HAL_Driver\Inc` 是否在里面
3. 用 `.\` 相对路径，不要用绝对路径

### Q2: 编译错误 `FreeRTOS.h: No such file or directory`

**原因**：FreeRTOS 头文件路径没配好。

**解决**：
1. 确认 FreeRTOS 源码在 `Middlewares\Third_Party\FreeRTOS\Source\` 下面
2. Include Paths 添加 `.\Middlewares\Third_Party\FreeRTOS\Source\include`

### Q3: 编译错误 `cmsis_os2.h: No such file or directory`

**原因**：CMSIS-FreeRTOS 适配层没添加。

**解决**：
1. Include Paths 添加 `.\Middlewares\Third_Party\FreeRTOS\Source\CMSIS_RTOS_V2`
2. 确保 `cmsis_os2.c` 在 FreeRTOS_Core 组中

### Q4: 编译错误 `undefined reference to HAL_TIM_PWM_Init`

**原因**：忘了添加对应的 HAL 源文件。

**解决**：把 `stm32f4xx_hal_tim.c` 和 `stm32f4xx_hal_tim_ex.c` 加到 HAL_Driver 组。

### Q5: 编译错误 `FreeRTOSConfig.h not found`

我们的 `FreeRTOSConfig.h` 在 `Inc/` 目录下，FreeRTOS 的 `FreeRTOS.h` 会 include 它。如果报错：

1. 确认 `.\Inc` 在 Include Paths 中
2. 这个路径**必须在** FreeRTOS 头文件路径**之前**（优先搜索）

### Q6: 下载后机器人不动/电机不转

排查步骤：
1. 看 OLED 有没有显示 → 判断 STM32 是否在运行
2. 用万用表量电机驱动板的 12V 供电是否正常
3. 在 `main.c` 中，初始化完成后加一个蜂鸣器测试音
4. 在调试模式中看 `g_motors[0].current_rpm` 是否变化

### Q7: MaixCAM 没反应/STM32 收不到数据

排查步骤：
1. 用示波器或逻辑分析仪测 UART TX 引脚有无信号
2. 检查 MaixCAM 和 STM32 是否共地
3. 波特率是否两边都设为 115200
4. TX ↔ RX 是否接交叉了（MaixCAM TX 接 STM32 RX）

### Q8: 机械臂动作角度不对

排查步骤：
1. 确认舵机型号和参数匹配（不同舵机角度范围不同）
2. `servo.c` 中的 `pulse_min` / `pulse_max` 是否对应舵机规格
3. 机械安装时是否已经校准好零位
